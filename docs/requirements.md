## 1. プロジェクト概要

### 1.1 プロジェクト名

肌ナビ（HadaNavi）― あなたの肌に合うプチプラコスメ、見つかる。

### 1.2 概要

肌タイプ診断(5問)・肌悩み選択・予算指定の3ステップで、最適なプチプラスキンケア商品を提案するWebアプリ。「ブランド横断×スキンケア手順×商品リンク」のセット提案が差別化軸。

### 1.3 背景

- プチプラコスメ市場約7,446億円。初心者は「何を買えばいいか」「手順がわからない」問題を持つ
- 既存サービスは口コミランキング主体。肌悩み×手順×予算統合型が不在
- 大手メーカーは自社商品誘導上ブランド横断不可（構造的参入障壁）

### 1.4 方針

- **MVP:** 静的サイト+診断フォーム+SEOコンテンツ
- **提案ロジック:** ルールベース決定木(API不要)
- **収益:** アフィリエイト(Amazon/楽天/Yahoo)+AdSense+「近くのドラッグストアで探す」ボタン
- **開発:** 個人開発(ソロ)

## 3. 機能要件

### 3.1 肌タイプ診断

| 肌タイプ | ID | 概要 |
|----------|-----|------|
| 乾燥肌 | dry | 水分・油分不足。バリア機能低下 |
| 脂性肌 | oily | 皮脂活発。毛穴・ニキビ |
| 混合肌 | combination | Tゾーン脂性/Uゾーン乾燥 |
| 敏感肌 | sensitive | バリア低下。赤み・ピリつき |
| 普通肌 | normal | バランス良好 |

### 3.2 肌悩み選択（8種類・複数可）

| 悩みID | 名称 | 美容液 | safe数 |
|--------|------|--------|--------|
| dryness | 乾燥 | 2 | 2 |
| pores | 毛穴 | 3 | 0 |
| acne | ニキビ | 3 | 1 |
| dullness | くすみ | 3 | 1 |
| spots | シミ | 2 | 1 |
| aging | ハリ不足 | 2 | 2 |
| redness | 赤み | 2 | 2 |
| oiliness | テカリ | 2 | 0 |

> ⚠ poresとoilinessはsensitivity_safe=trueが0件。敏感肌ユーザーにはsensitive_alternativesで代替提案する。

### 3.3 予算フィルター

| 予算帯 | ラベル | 上限 | 説明 |
|--------|--------|------|------|
| under_1000 | 〜1,000円/月 | ¥800 | 3ステップ |
| under_3000 | 〜3,000円/月 | ¥1,500 | 5ステップ |
| under_5000 | 〜5,000円/月 | ¥2,500 | 美容液追加 |
| no_limit | 予算気にしない | — | プチプラ内最適 |

#### フォールバックルール

- **Rule1:** カテゴリ0件→最安商品1件+「予算を少し超えますが」注記
- **Rule2:** sensitive+under_1000→max1.5倍緩和(¥800→¥1,200)
- **Rule3:** 全ステップ不可→minimal_routine(3ステップ)優先

### 3.4 季節アドバイス

| 季節 | 悩み | 推奨成分 |
|------|------|----------|
| 春(3-5月) | 花粉肌荒れ | セラミド・CICA |
| 夏(6-8月) | テカリ/UV | ビタミンC・ナイアシンアミド |
| 秋(9-11月) | 夏ダメージ回復 | レチノール・セラミド |
| 冬(12-2月) | 乾燥・バリア低下 | セラミド・スクワラン |

### 4.1 商品データフィールド

| フィールド | 型 | 説明 |
|------------|-----|------|
| product_id | string | ユニーク識別子(重複排除用) |
| name/brand/price | — | 商品名・ブランド・税込価格 |
| key_point | string | 提案理由(薬機法準拠) |

| ステップ | 商品数 | データキー |
|----------|--------|------------|
| cleansing | 14 | skin_type別 |
| face_wash | 14 | skin_type別 |
| toner | 14 | skin_type別 |
| serum | 26 | concern別(最多) |
| moisturizer | 14 | skin_type別 |
| sunscreen | 10 | skin_type別 |

### 4.3 ビルド時前処理

- **商品インデックス:** product_idでユニーク化したフラット配列+出現ステップ×肌タイプ/悩みのインデックス構築

## 5. 診断ロジック・スコアリングマトリクス

### 5.1 診断フロー

STEP1: 肌タイプ診断(5問・クライアントJS完結) → STEP2: 肌悩み選択(複数可) → STEP3: 予算選択(4段階) → 結果

### 5.2 スコアリングマトリクス（全質問定義済み）

各回答に肌タイプ別重み付けスコアを加算。最高スコアの肌タイプを判定。
理論最大: dry=11, oily=11, combination=11, sensitive=7, normal=12

#### Q1: 洗顔後10分放置した肌の状態は？

| | 乾燥 | 脂性 | 混合 | 敏感 | 普通 |
|---|---|---|---|---|---|
| 全体つっぱる | 3 | 0 | 0 | 1 | 0 |
| 全体ベタつく | 0 | 3 | 0 | 0 | 0 |
| Tゾーンベタつき頬つっぱる | 0 | 0 | 3 | 0 | 0 |
| 特に変化なし | 0 | 0 | 0 | 0 | 3 |

#### Q2: 日中のメイク崩れは？

| | 乾燥 | 脂性 | 混合 | 敏感 | 普通 |
|---|---|---|---|---|---|
| FD割れ・粉っぽい | 3 | 0 | 1 | 1 | 0 |
| Tゾーンドロドロ | 0 | 3 | 2 | 0 | 0 |
| 鼻テカり頬乾く | 0 | 0 | 3 | 0 | 0 |
| あまり崩れない | 0 | 0 | 0 | 0 | 3 |

#### Q3: 新しい化粧品で刺激を感じる？

| | 乾燥 | 脂性 | 混合 | 敏感 | 普通 |
|---|---|---|---|---|---|
| よくある(赤み・ピリつき) | 0 | 0 | 0 | 3 | 0 |
| たまにある | 1 | 0 | 0 | 1 | 0 |
| ほとんどない | 0 | 0 | 0 | 0 | 1 |

#### Q4: 毛穴の状態で気になるのは？

| | 乾燥 | 脂性 | 混合 | 敏感 | 普通 |
|---|---|---|---|---|---|
| 目立たないが乾燥小ジワ | 3 | 0 | 0 | 1 | 0 |
| 鼻・頬の毛穴が開き黒ずむ | 0 | 3 | 2 | 0 | 0 |
| Tゾーンだけ目立つ | 0 | 1 | 3 | 0 | 0 |
| 特に気にならない | 0 | 0 | 0 | 0 | 3 |

#### Q5: 季節の変わり目に肌が変わる？

| | 乾燥 | 脂性 | 混合 | 敏感 | 普通 |
|---|---|---|---|---|---|
| 冬にカサカサ・粉ふき | 2 | 0 | 0 | 1 | 0 |
| 夏にベタつき・ニキビ増 | 0 | 2 | 1 | 0 | 0 |
| 季節ごとに乾燥もテカリも変わる | 0 | 0 | 2 | 1 | 0 |
| あまり変わらない | 0 | 0 | 0 | 0 | 2 |

### 5.3 同点判定（tie_breaking）

優先順: **sensitive > dry > combination > oily > normal**

理由: sensitiveは安全性直結で最優先。「あなたは○○肌寄りの△△肌です」と両方表示も可。

### 5.4 スキンケアルーティン

| ルーティン | ステップ |
|------------|----------|
| 朝 | 洗顔→化粧水→美容液→乳液→日焼け止め |
| 夜 | クレンジング→洗顔→化粧水→美容液→乳液 |
| 最小限 | 洗顔→化粧水→乳液(3ステップ) |

## 6. 商品レコメンドロジック・優先順位

### 6.1 フィルタリングフロー

1. **Step1:** 肌タイプ別にクレンジング〜乳液・日焼け止め選定
2. **Step2:** 肌悩み(concern)別に美容液選定
3. **Step3:** 安全性フィルター(sensitive→sensitivity_safe=false除外)
4. **Step4:** exclude_skin_typesフィルター
5. **Step5:** 予算フィルター+フォールバック
6. **Step6:** 濃度注意書き表示

### 6.2 表示優先順位

1. **1位:** sensitivity_safe=trueを優先
2. **2位:** 価格が安い順
3. **3位:** evidence_level=医薬部外品を優先

### 6.3 重複商品

product_idベースで重複排除、結果画面で1回のみ表示。

## 7. 安全性・薬機法対応

### 7.1 敏感肌安全性フィルター

全カテゴリでsensitivity_safe=true商品存在確認済。全診断パターンで0件にならないこと検証済。

### 7.2 薬機法表現ガイド

**許容(9種):** 肌を整える / うるおいを与える / キメを整える / なめらかにする / 清浄にする / ツヤを与える / ハリを与える / すこやかに保つ / 引き締める

**禁止:** シワ改善 / 美白 / ニキビを治す / シミが消える / アンチエイジング / 若返り

**医薬部外品追加許容(4種):** メラニン抑制/シワ改善(認可成分)/肌あれ防止/日やけによるしみ防止

## 8. 結果表示・情報設計・OGP動的生成

### 8.1 段階表示ロジック

- **デフォルト:** 最小限ルーティン3ステップ(洗顔・化粧水・乳液)の提案商品を表示
- **「フルルーティンを見る」ボタン:** クリックでクレンジング・美容液・日焼け止めも展開
- **under_1000:** minimal_routineのみ。「余裕ができたら追加」表示

### 8.2 商品カード

商品画像 + 名前 + ブランド + 価格 + key_point(成分説明) + アフィリボタン(Amazon/楽天/「近くのドラッグストア」)

### 8.3 OGP動的生成（フェーズ1必須）

- **技術:** Next.js API Routes + @vercel/og
- **テンプレ:** 「私の肌タイプは○○肌！おすすめはこれ！」+ 肌ナビロゴ
- **URL:** `/api/og?type=dry&concern=pores&budget=under_3000`
- **サイズ:** 1200x630px (Twitter/Facebook/LINE)
